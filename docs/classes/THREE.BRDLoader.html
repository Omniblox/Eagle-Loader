Defined in src/BRDLoader.js:137

Construct the EagleBrdRenderer object THREE.js geometry (brd.root), bearing composited textures from all layers.

Parameters:

  • data String

    Contents of the .brd file as a string

  • [params] Object optional

    Composite parameter object

    • [colors] Object optional

      Define custom colors; see this.colors

    • [composite=true] Boolean optional

      Whether to composite layers, or render them as individual geometries. Warning: individual layers are very slow.

    • [maskOpacity=0.8] Number optional

      Opacity of solder mask; opacity is halved over copper traces

    • [material="phong"] String optional

      Material shader to use. Options include "phong" for realistic lighting, "lambert" for flat lighting, and "basic" for no lighting.

    • [pixelMicrons=35] Number optional

      Resolution of texture maps. By default, this is 35 microns, equal to the thickness of a default copper layer. Note that this will affect the size of the board geometry.

    • [thickness] Number optional

      Override computed thickness of board. Measured in millimeters. Note that this value will be converted to pixels for use within the board.

    • [viewConnectors=false] Boolean optional

      Whether to visualize Connector objects

    • [viewGhosts=false] Boolean optional

      Whether to draw approximate ghosts of on-board devices

    • [viewComponents=false] Boolean optional

      Whether to load and display models of the components on the PCB.

    • [componentMapCfg] Object optional

      Alternative standard model library map configuration to use instead of the default.

      • [mapUrl] String optional
        URL for the file that contains the "component package name" to "model file path" mapping.
      • [urlPrefix] String optional
        (Optional) URL prefix to prepend to the model file path when loading model file.

        (Optional) URL prefix to prepend to the model file path when loading model file.
  • onLoad Function

    Will be called when load completes. The argument will be the loaded Object3D.

  • [onProgress] Function optional

    Will be called while load progresses. The argument will be the XmlHttpRequest instance, that contain .total and .loaded bytes.

  • [onError] Function optional

    Will be called when load errors.

  • [fontPath] Array optional

    Paths to font files. Defaults to the array "./OCRA.woff", "./OCRA.otf". You may specify any number of files as fallbacks, but a WOFF and OTF should cover all your bases. If you specify null, the loader will skip font loading.
